Background

Read the quick guide to Prolog and the comments in test.pl (propositional logic) and test2.pl (predicate logic). before beginning the lab. You may also find the overview of the Prolog language in ch. 16.4 - 16.6.6 of the text helpful. This documentation will also be useful for the lab on Prolog (22 pg. PDF). If your logic is a bit rusty, see propositional logic and predicate logic.

The file test.pl contains facts and rules for constructing a logic-based a family tree. To experiment with Prolog, copy test.pl into your directory: 

    cp /home/fac/donna/public_html/cs350/Code/Prolog/test.pl  .
Start the interactive Prolog interpreter: 
    prolog
Load and compile test.pl into bytecode: 
    |-? [test].
List the facts and rules: 
    |-? listing.
Execute these three queries: 
    |-? female(X). 

    |-? closeRelative(elizabeth,ken).

    |-? ancestor(irvin,Y).
Confirm that the queries return what you think they should return. Note that if a rule on a single line contains ',' AND and ';' OR, AND takes precedence over OR.

Hit CTRL-D to exit the Prolog interpreter.

For more work on predicate logic, copy test2.pl from /Code/Prolog into your directory. Follow the instructions in the file.

No need to submit anything for Part I - this is just to get you used to Prolog syntax.

PART I

Copy this file into your directory:
      cp /home/fac/donna/public_html/cs350/lab04a.pl .
You job is to is convert the arguments in lab04a.pl into propositional logic following the syntax of Prolog rules and facts. You are, in essense, constructing an inference engine of simple propositional logic. (Prolog is really based on predicate logic, but propositional logic will introduce you to how the inference engine works.)

Follow the instructions in the file. If your engine is working correctly you should be able to enter the queries (argument conclusions) as requested in the file at the interactive prompt. If the argument is valid, Prolog will respond 'yes.' If the argument is a fallacy, Prolog will respond 'no.'

PART II

The goal of this part of the lab is to write a prolog program that will apply inferences rules to arguments based on predicate logic. All you need to do is convert the arguments listed in this file into Prolog rules and facts. First copy the file into your directory:

Follow the instructions in the file.
